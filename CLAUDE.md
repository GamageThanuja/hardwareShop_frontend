## Architecture

**Stack:** Next.js (App Router) + React 19 + TypeScript + Tailwind CSS v4 + Shadcn UI + Zustand

This is a multi-dashboard UI template with no backend. All data is mocked/static.

### App Router structure

`src/app/` uses Next.js App Router. `/dashboard` redirects to `/dashboard/default`. There are multiple dashboard variants (default, crm, finance, analytics, productivity, ecommerce, academy), each a separate route under `src/app/dashboard/`.

### Preferences & Theme system

Global UI preferences (theme mode, color preset, font, layout options) are stored in a Zustand store (`src/stores/preferences/`) and persisted to client cookies. On mount, the provider syncs preferences to DOM `data-*` attributes on `<html>` for SSR hydration safety. A `ThemeBootScript` injected in the root layout prevents theme flicker.

Available preferences:
- `theme_mode`: `"light" | "dark" | "system"`
- `theme_preset`: `"default" | "brutalist" | "soft-pop" | "tangerine"`
- `font`: 14 options (Geist, Inter, Roboto, etc.)
- `content_layout`: `"centered" | "full-width"`
- `navbar_style`: `"sticky" | "scroll"`
- `sidebar_variant`: `"sidebar" | "inset" | "floating"`
- `sidebar_collapsible`: `"icon" | "offcanvas"`

### Theme presets

CSS preset files live in `src/styles/presets/` (e.g. `brutalist.css`). The script `src/scripts/generate-theme-presets.ts` parses these files and outputs `src/lib/preferences/theme.ts` with metadata. **Never edit `theme.ts` directly** — it is auto-generated by `npm run generate:presets`, which also runs automatically via the Husky pre-commit hook.

Colors use OkLCH format. Dark mode is implemented via `@custom-variant dark (&:is(.dark *))` in `globals.css` — not via Tailwind's built-in dark mode.

### Tailwind config

There is no `tailwind.config.ts`. Tailwind v4 is configured inline in `src/app/globals.css` using `@theme` and `@custom-variant` directives.

### Components

`src/components/ui/` holds Shadcn-based primitives (excluded from Biome linting). Feature components live colocated with their routes or in `src/components/`. Use `cn()` from `src/lib/utils.ts` for class merging, CVA for variant styling.

### State management

Zustand stores are in `src/stores/`. The preferences store uses a vanilla Zustand store distributed via React context (not the default module-singleton pattern) to support SSR.

### Path alias

`@/*` maps to `src/*`.

## Linting & formatting

Biome replaces ESLint and Prettier. Line width: 120, indent: 2 spaces, quotes: double. Import order is enforced: react → next → packages → `@/*` aliases → relative paths. Run `npm run check:fix` to auto-fix.
